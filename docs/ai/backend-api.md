# Backend API
## Axios:
Es una librería de JavaScript que permite hacer peticiones HTTP desde el navegador o desde Node.js.
Se utiliza principalmente para comunicarse con APIs, enviando y recibiendo datos de forma sencilla (GET, POST, PUT, DELETE, etc.).

¿Por qué se usa?
- Facilita la comunicación entre frontend y backend
- Permite manejar respuestas y errores fácilmente
- Es más cómodo y potente que usar fetch directamente

## Postman: 
Es una herramienta que permite probar APIs sin necesidad de usar el frontend.
Se puede enviar peticiones HTTP y ver las respuestas del servidor de forma clara.

¿Por qué se usa?
- Para comprobar que la API funciona correctamente
- Para probar endpoints (GET, POST, DELETE…)
- Para depurar errores en el backend

## Sentry
Es una plataforma de monitorización de errores en aplicaciones.
Permite detectar fallos en tiempo real y ver exactamente qué ha pasado.

¿Por qué se usa?
- Para detectar errores automáticamente
- Para obtener información detallada de los fallos
- Para mejorar la estabilidad de la aplicación

## Swagger
Es una herramienta para documentar APIs.

Permite crear documentación interactiva donde se pueden ver y probar los endpoints.

¿Por qué se usa?
- Para tener una documentación clara de la API
- Para que otros desarrolladores entiendan cómo usarla
- Para probar endpoints directamente desde la documentación
